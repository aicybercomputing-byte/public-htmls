# Azure Function Job Search Spec (Full Parity)

This spec implements runtime parity with the legacy Google Apps Script job-search API while Power Automate handles scheduled ingestion.

Service account context:

- Power Platform/Azure automation owner: `bellini-webmaster@usf.edu`

## Scope

- Runtime endpoints:
  - `GET /api/health`
  - `POST /api/health`
  - `GET /api/job-search`
  - `POST /api/job-search`
- Behaviors to preserve:
  - strict request validation
  - optional API key gate
  - per-hour rate limiting
  - query cache
  - provider fan-out (USAJOBS + Simplify)
  - dedupe + ranking + per-provider cap
  - JSON output parity
  - optional JSONP callback support for GET

## Recommended stack

- Azure Functions (Node.js 20 or Python 3.11)
- Azure Cache for Redis (rate limit + query cache)
- Azure Blob Storage (provider snapshots from Flow)
- Application Insights (telemetry)

## Routes and contract

## `GET|POST /api/health`

Response:

```json
{"ok": true, "service": "jobs-api-azure"}
```

## `GET|POST /api/job-search`

Supported fields:

- `query_text` (required)
- `location_text` (required)
- `min_pay` (optional number)
- `employment_type` (optional string)
- `remote_mode` (optional string)
- `limit_per_provider` (optional integer 1..20, default 8)
- `notes` (optional string)
- `authorization` (optional string)
- `api_key` (optional/required depending on config)
- `callback` (optional GET-only JSONP callback)

Success response:

```json
{
  "request_id": "uuid",
  "cache_hit": false,
  "results": [],
  "provider_status": []
}
```

Error body conventions:

- `{"error":"Bad request"}`
- `{"error":"Unauthorized"}`
- `{"error":"Rate limit exceeded"}`
- `{"error":"Internal error"}`

## Environment variables

- `WEBAPP_API_KEY` (optional)
- `RATE_LIMIT_PER_HOUR` (default `400`)
- `QUERY_CACHE_TTL_SEC` (default `21600`)
- `USAJOBS_API_KEY`
- `USAJOBS_USER_AGENT`
- `BLOB_ACCOUNT_URL`
- `BLOB_CONTAINER_NAME`
- `BLOB_PATH_USAJOBS` (e.g. `job-search/providers/usajobs/latest.json`)
- `BLOB_PATH_SIMPLIFY` (e.g. `job-search/providers/simplify/latest.json`)
- `ENABLE_JSONP` (`true|false`, default `false`)

## Request pipeline

1. Parse request (query for GET, JSON body for POST).
2. Validate allowlisted keys and bounds.
3. Enforce API key if configured.
4. Enforce rate limit in Redis:
   - key: `rl:{hash(api_key_or_anon|hour_bucket)}`
   - operation: `INCR` + `EXPIRE 3600`
5. Build normalized query hash.
6. Query Redis cache:
   - key: `query:{sha256(normalized_payload)}`
   - on hit return cached payload with `cache_hit: true`
7. Load provider data:
   - from Blob snapshots generated by Power Automate, or
   - live fallback HTTP calls when snapshots unavailable
8. Normalize to common row shape.
9. Dedupe, score, cap/interleave per provider.
10. Cache payload in Redis (TTL from env).
11. Return JSON (or JSONP if enabled + valid callback).

## Provider normalization shape

Each row:

- `result_id`
- `title`
- `location`
- `pay`
- `apply_url`
- `provider`
- `match_score`
- `expiry_at`
- `source`

## Security controls

- Validate callback name against strict regex:
  - `^[a-zA-Z_$][0-9a-zA-Z_$]*$`
- Reject unknown fields (`allowlist` validation).
- Reject oversized payloads (max body bytes).
- Do not log secrets or full raw request bodies.
- Store secrets in Key Vault or secured app settings.

## Testing matrix

- Health route status/body.
- GET and POST parity for search.
- Validation failures for missing required fields.
- API key mismatch returns Unauthorized.
- Rate-limit threshold returns expected error.
- Cache hit path sets `cache_hit: true`.
- Provider partial failure still returns usable results with `provider_status` errors.
- JSONP enabled/disabled behavior.

## Deployment order

1. Deploy Function App (staging).
2. Configure env vars + managed identity.
3. Configure Redis + Blob access.
4. Run smoke tests.
5. Point front-end to new endpoint.
6. Monitor logs and latency; roll back URL if needed.
