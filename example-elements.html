USF.edu element library

Real, on-brand elements copied verbatim from usf.edu's homepage (captured 2026-07-13) — a library to hand to future agents building new pages, so they can copy real usf.edu markup instead of guessing at class names or reinventing components. "View markup" under each example is auto-extracted from the live DOM, so it always matches exactly what's rendered.

Every class below (.button, .card, .section--*, .hero__*, .stat, …) is usf.edu's own production class, part of their v4 design system (tokens.css, base-utilities.css, ui-components.css, styles.css — all four vendored locally at reference/usf-cms/css/v4/). Copy these classes as-is into a real page; never rename or restyle them, the same rule this repo already applies to CMS theme classes like c-calloutText (see tcap/omnicms-snippets/NAMING-CONVENTIONS.md, rule 2). Some interactivity (dropdown toggles, the video dialog, mobile drawer) needs usf.edu's own JS, which isn't loaded here — dropdowns below are shown pre-expanded for reference instead.

The raw, unedited page source this was distilled from is saved at reference/usf-cms/tmp-usf-homepage.html — re-extract from the live site (not from that stale copy) if usf.edu has since redesigned.

Section color system

Every homepage content block is a .section with a background modifier (--white, --sand, --usfgreen, --evergreen) plus --dark for light-on-dark text, laid out with .section__inner.grid.grid-col-12-sm and populated with .section__media / .section__text / .section__cta. Other modifiers seen on the real homepage: section--connected-lines-left, section--gradient-left, section--top-desc-tint, section--top/bottom-desc-solid, section--image with section--image-contain-bl / -cover-cc / -overlay.
