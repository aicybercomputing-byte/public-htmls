OmniCMS element library

Real, on-brand elements for future agents building OmniCMS pages: usf.edu production patterns captured from usf.edu's homepage on 2026-07-13, plus this repo's own reusable snip-* component system. "View markup" under each example is auto-extracted from the live DOM, so it always matches exactly what's rendered.

USF classes below (.button, .card, .section--*, .hero__*, .stat, ...) are usf.edu's own production classes, part of their v4 design system (tokens.css, base-utilities.css, ui-components.css, styles.css - all four vendored locally at reference/usf-cms/css/v4/). Copy USF and CMS theme classes as-is into a real page; never rename or restyle them, the same rule this repo already applies to CMS theme classes like c-calloutText (see tcap/omnicms-snippets/NAMING-CONVENTIONS.md, rule 2). Some interactivity (dropdown toggles, the video dialog, mobile drawer) needs usf.edu's own JS, which isn't loaded here - dropdowns below are shown pre-expanded for reference instead.

The raw, unedited USF page source is saved at reference/usf-cms/tmp-usf-homepage.html - re-extract from the live site (not from that stale copy) if usf.edu has since redesigned.

This merged catalog includes both usf.edu production elements and this repo's own OmniCMS snippet component system (the snip-* classes), sorted by element type so agents can scan from page structure to actions, content blocks, data displays, and interactive widgets.

Section color system
