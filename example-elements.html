USF.edu element library

Real, on-brand elements copied verbatim from usf.edu's homepage (captured 2026-07-13) — a library to hand to future agents building new pages, so they can copy real usf.edu markup instead of guessing at class names or reinventing components. "View markup" under each example is auto-extracted from the live DOM, so it always matches exactly what's rendered.

Every class below (.button, .card, .section--*, .hero__*, .stat, …) is usf.edu's own production class, part of their v4 design system (tokens.css, base-utilities.css, ui-components.css, styles.css — all four vendored locally at reference/usf-cms/css/v4/). Copy these classes as-is into a real page; never rename or restyle them, the same rule this repo already applies to CMS theme classes like c-calloutText (see tcap/omnicms-snippets/NAMING-CONVENTIONS.md, rule 2). Some interactivity (dropdown toggles, the video dialog, mobile drawer) needs usf.edu's own JS, which isn't loaded here — dropdowns below are shown pre-expanded for reference instead.

The raw, unedited page source this was distilled from is saved at reference/usf-cms/tmp-usf-homepage.html — re-extract from the live site (not from that stale copy) if usf.edu has since redesigned.

This page holds two libraries: the usf.edu homepage elements above, and this repo's own OmniCMS snippet component system (the snip-* classes) appended at the bottom ↓.

OmniCMS snippet component system (.snip-*)

The second library on this page: this repo's own namespaced, re-themeable component system for OmniCMS content snippets — every reusable pattern the four *.snippet.html files share, under one .snip-* name each. Unlike the usf.edu elements above (which are usf.edu's live production classes), these are ours: copy the markup from any "View markup" panel, and re-skin by overriding a handful of CSS custom properties. Styling comes from snippet-template.css, behavior from snippet-template.js.

Full rules in NAMING-CONVENTIONS.md; a dense, copy-paste-first version for generating new snippets in CATALOG.md. When building a page that gets pasted into OmniCMS, don't link these files the way this reference page does — the CMS strips external <link>/<script src> tags on save; inline the CSS/JS instead (see CATALOG.md → "Quick start").

Third-party & generated (documented, not templated)

These exist in the CMS today but aren't part of the snip- system — either the markup belongs to a third-party library, or it's built entirely by JS with no CSS class API to standardize against. Same rule as never renaming the CMS theme's own classes.

carosels/news/2026/website_story.html — a Bootstrap 3 image carousel embed. Every class (carousel, item, carousel-control) is Bootstrap's own; used as-is via the Bootstrap CDN.
ai-x/all-events.html & ai-x/ai-x-past-events.html — self-contained JS widgets that build their DOM with inline style.cssText rather than CSS classes (year/month/event archive tree from a CSV). No class API exists to fold into this system.

snip-* components generated from snippet-template.css; mirror of tcap/omnicms-snippets/style-guide.html. "View markup" panels are auto-extracted from the live DOM, so they always match what's rendered.
